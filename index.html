Description

DuinoMultiRemote

Est un plugin de démonstration permettant à S.A.R.A.H d'interagir avec une carte Arduino.
Afin d'évaluer les possibilités d'interactions offertes, quelques composants sont cablés sur la breadbord et permettent de "visualiser" les actions.

Attention: Il vous faut utiliser impérativement la version V4 de S.A.R.A.H

Les communications sont possibles depuis:
    Le portlet du plugin:
      - Envoi des commandes directement depuis l'écran.
      - Retour d'action avec mise à jour en temps réel.
    Les composants de l'Arduino
      - Switch.
      - Potentiomètre.
    La commande vocale intégrée.
L'ensemble vous permettant de commander:
    Une led RGB:
      - On/Off et changement de couleur.
    Une led Normale:
      - On/Off et variation.
    Un afficheur LCD:
      - Etat du plugin (Ready/OFF).
      - Valeurs de la LED RGB (RGB(x,x,x).
      - Valeur de la LED dimmable (LED(x)).
      - Etat des LED's (ON/OFF)

La plupart des montages à base d'Arduino, sont réalisés dans un but pédagogique. Ce plugin relève de cette même volontée et devrait vous permettre d'évaluer le potentiel des possibilités des communications "real-time" entre l'Arduino et S.A.R.A.H.

Mais il peut aussi s'adapter - facilement - à vos besoins spécifiques d'IOT/Domotique.

Code
DuinoMultiRemote utilise les librairies NodeJS suivantes :
  • node-serialport - Pour les communications entre le plugin et l'arduino.
  • socket.IO - Pour la liaison "real-time" entre le plugin et son portlet.
Mais aussi :
  • spectrum - Pour la gestion d'un colorpicker dans le portlet.
Principe
Chronologie :

    1 - Les requètes sont envoyées vers l'Arduino via la librairie serialPort depuis le port COM de l'ordinateur.
    2 - Le sketch de l'Arduino retourne (toujours via serialPort), le nouvel état de ses composants au plugin et met à jour l'afficheur LCD.
    3 - La librairie socket.IO permet de mettre à jour immédiatement le portlet afin de refleter le résultat de la requète.

Schéma fonctionnel (principe):

Arduino SerialPort Plugin Socket.IO Portlet

Avantages:

    socketIo évite de rafraîchir le navigateur, ou de mettre en place un CRON ou un timer pour mettre à jour le portlet.
    serialPort permet la communication série USB avec l'Arduino.

Port COM

Pour pouvoir fonctionner, le module a besoin de connaître le port COM{x} de l'Arduino afin de lui envoyer les requêtes demandées via la liaison USB. (Il est à noter que la gestion via des shields "ethernet" est aussi possible...).

Vous devez donc, via le portlet du module, paramétrer le port COM :

  • Port : {COM{x}}

Ou {x} représente le numéro du port. (ie: COM3)

DuinoMultiRemote vous avertira des erreurs éventuelles dans la console du serveur.

(Vous aurez peut-être besoin de relancer SARAH aprés cette action...)

Programmation Arduino

Pour pouvoir fonctionner correctement avec DuinoMultiRemote, l'Arduino à besoin du programme adapté ou sketch.

Ce programme est disponible dans le sous-répertoire /arduino/multiduinoremote du plugin sous le nom DuinoMultiRemote.ino

Vous devrez donc impérativement le compiler, puis le "téléverser" avec un programme adapté dans la mémoire de l'Arduino, afin d'exploiter les possibiltés offertes par ce plugin...

Grammaire

L'ensemble de la grammaire utilisée par le module est stockée dans le fichier DuinoRemote.xml

Ce fichier est aisément éditable par vos soins pour vous permettre de modifier, ajouter ou retirer une commande vocale.

Commandes vocales
Les commandes sont divisées en 2 catégories :
  • Commandes d'action.
  • Commandes de retour d'état.
Les commandes d'action permettent de modifier l'état d'un périphérique et les commandes de retour d'état permettent d'en connaître le statut.

Il est à noter que si une commande ne peut aboutir, le plugin vous avertira :

Exemple: Sarah allume la Led alors que la LED est déjà allumée... S.A.R.A.H vous répondra : La LED est déjà allumée.

Exemples de commandes vocales
Commandes d'action :
(Exemples) 

SARAH allume la LED

  • Envoi "L{ledValue},1\n" sur le port COM
  • Retour vocal: La Led est allumée
SARAH éteinds la LED
  • Envoi "L{ledValue},0\n" sur le port COM
  • Retour vocal: La Led est éteinte
SARAH éteinds la RGB
  • Envoi "R{red-val},{green-val},{blue-val},0\n" sur le port COM
  • Retour vocal: La RGB est éteinte
SARAH allume la RGB
  • Envoi "R{red-val},{green-val},{blue-val},1\n" sur le port COM
  • Retour vocal: La RGB est allumée
Commande de retour d'état:
(Exemple) 

SARAH quel est l'état de la LED

  • Envoi "?\n" sur le port COM
  • Retour vocal: La Led est {éteinte,allumée}

Commandes Arduino

Il est possible de commander les LED's via les 2 composants "actifs" cablés sur la breadboard.

1 - Le potentiomètre :
  • Est relié via le sketch Arduino à la LED.
  • Une rotation permet de faire varier l'intensité lumineuse de la LED.
  • La valeur analogique du potentiomètre est mise à jour dans le portlet.
2 - Le bouton poussoir :
  • Est relié via le sketch Arduino à la LED et à la RGB.
  • Un appui bref permet d'éteindre/ allumer la LED et la RGB.
  • Mise à jour de l'état et des valeurs d'intensité des LED's dans le portlet.
  • Affichage du nombre de cycles d'allumage/extinction.
Un second potentiomètre est présent sur la breadboard de l'Arduino...
Ce dernier n'est pas lié à SARAH, ni à l'Arduino mais directement à l'afficheur LCD. Il vous permettra, par sa rotation, d'en regler le contraste afin de rendre l'affichage clairement visible.

Commandes "Portlet"

Le portlet étant relié à l'Arduino en mode bidirectionnel (via le plugin), il est tout à fait possible d'utiliser votre écran pour envoyer vos commandes et pour en visualiser les retour. La mise à jour du portlet étant quasi instantanée...

Vous pouvez donc clicker sur :

  • Les boutons actifs : ON  pour en changer l'état   OFF

  • Les sliders :
      Pour en faire varier la valeur.
Les requètes correspondantes sont alors immédiatement envoyées à l'Arduino et le portlet se met immédiatement à jour... Toutefois, les commandes du portlet n'étant pas des commandes vocales, aucun retour ne sera vocalisé par S.A.R.A.H., seul le portlet affichera le résultat de l'action demandée.
Erreurs
Les erreurs générées par le programme peuvent notament, avoir les origines suivantes:
  • L'Arduino n'est pas branché avant le lancement du serveur.
  • Erreur de paramètrage du port COM.
  • Erreur lors de la requète.
La gestion de ces erreurs n'est pas totalement implémentée sur ce plugin...
Il vous appartiendra de les traiter dans votre plugin définitif.
